# Task 14: Linux Server Hardening & Secure Configuration

## Objective
To harden a Linux system by applying security best practices and reducing the attack surface.

## System Used
- Kali Linux

## Hardening Steps Performed
- Reviewed system users, services, and open ports
- Verified no unused human user accounts
- Restricted sudo access using least privilege
- Disabled root login over SSH
- Configured SSH key-based authentication
- Reviewed system updates and handled dependencies safely
- Configured UFW firewall with restrictive rules
- Stopped and disabled unnecessary services
- Hardened permissions for sensitive system files
- Reviewed authentication and system logs

## Deliverables
- Linux Hardening Checklist
- Security Configuration Summary

## Final Outcome
This task improved Linux system security by enforcing access controls, minimizing exposed services, and applying secure configurations.
